# План обширной админ-панели Caramel

Чтобы не править код вручную, всё управление контентом — из одной панели. Ниже перечень разделов и возможностей.

---

## 1. Меню (страница menu.html)

| Что | Возможности |
|-----|--------------|
| **Позиции** | Список всех блюд/напитков с фильтром по разделу (Еда / Напитки / Десерты / Выпечка) и категории (Кофе, Пицца, Блинчики и т.д.) |
| **Редактирование** | Для каждой позиции: название (RU, UZ, EN, TR), описание (4 языка), цена, группа, раздел, категория |
| **Доп. поля** | Путь к фото или «Без фото», объёмы для напитков (0.2, 0.3 л), метка «В порции X шт.» (для блинчиков и т.п.) |
| **Действия** | Добавить позицию, удалить позицию, **очистить всё меню** (удалить все позиции разом) |
| **Сохранение** | Кнопка «Сохранить меню» — изменения записываются в файл, сайт подхватывает новое меню |

---

## 2. Главная страница (index.html)

### 2.1 Hero (первый экран)
- Подзаголовок (RU, UZ, EN, TR), например: «Искусство вкуса с 1996 года»
- Текст кнопки «Открыть меню» (4 языка)
- Путь к фоновому видео (или отключить видео)

### 2.2 Блок «О нас» / История
- Заголовок и подзаголовок секции (4 языка)
- Подзаголовок «Семейная традиция с 1996 года» (4 языка)
- Два абзаца текста (p1, p2) — 4 языка
- Таймлайн: 2013 и 2020 — текст для каждого года (4 языка)
- Картинка блока (путь к файлу, например caramel_fon.png)

### 2.3 Отзывы
- Список отзывов: текст и автор (RU, UZ, EN, TR)
- Добавить отзыв / удалить отзыв / изменить порядок

### 2.4 Галерея
- Список слайдов: картинка (путь) + подпись (RU, UZ, EN, TR)
- Добавить слайд / удалить / изменить порядок

### 2.5 Локации и контакты
- Заголовок и подзаголовок секции (4 языка)
- **Филиал 1:** город, улица, часы работы, телефон
- **Филиал 2:** город, улица, часы работы, телефон
- Подписи «1 филиал», «2 филиал», «Часы работы», «Телефон» (4 языка)
- Ссылка на Instagram (URL и текст кнопки «Следите за нами…»)

### 2.6 Футер
- Текст «Все права защищены» (4 языка)
- Текст «Сделано Golden Pixel» (4 языка)
- Ссылка на разработчика (опционально)

---

## 3. Тексты и переводы (i18n)

- Все тексты с главной (навигация, hero, о нас, отзывы, галерея, локации, футер) хранятся в одном объекте переводов (RU, UZ, EN, TR).
- В админке: раздел «Тексты сайта» — дерево или список ключей (nav.home, hero.subtitle, about.title и т.д.) с полями для каждого языка.
- Сохранение — в один файл (например, `data/translations.json`), главная и меню подгружают переводы оттуда (или из API).

---

## 4. Настройки сайта

| Настройка | Описание |
|-----------|----------|
| Название сайта | Для заголовка вкладки и шапки (если выводится) |
| Логотип | Путь к файлу логотипа |
| Валюта | Обозначение (например, «сум» / «so'm») для меню |
| Пароль админки | Простой пароль для входа в админ-панель (опционально) |

---

## 5. Как это будет работать технически

1. **Сервер (Node.js)** — раздаёт сайт и отдаёт данные через API (меню, тексты, отзывы, галерея, настройки).
2. **Файлы данных** — например: `data/menu.json`, `data/home.json` (hero, о нас, локации, футер), `data/reviews.json`, `data/gallery.json`, `data/translations.json`, `data/settings.json`. Всё это админка читает и сохраняет.
3. **Админ-панель** — одна страница (например, `/admin`) с боковым меню: «Меню», «Главная», «Отзывы», «Галерея», «Локации», «Тексты», «Настройки». В каждом разделе — формы и кнопки сохранения.
4. **Сайт** — при открытии запрашивает данные с API и подставляет их в страницы (меню, главная). Если API недоступен (например, открыли index.html с диска), можно оставить текущее поведение: контент из встроенного в HTML (как сейчас).

---

## 6. Что не входит в базовый вариант (можно добавить позже)

- Загрузка новых изображений через админку (пока только указание пути к уже залитым в папку `images/` или в корень).
- Редактирование структуры разделов/категорий меню (списков кнопок «Еда», «Напитки» и т.д.) — при желании вынесем в настройки позже.
- Карта (координаты филиалов) — можно хранить в настройках локаций и подставлять в код карты.

---

## Итого: что будет «всё» в админке

- **Меню** — полное управление позициями (добавить / изменить / удалить / очистить всё).
- **Главная** — hero, о нас, отзывы, галерея, локации и контакты, футер.
- **Тексты** — все переводы (RU, UZ, EN, TR) в одном месте.
- **Настройки** — название, логотип, валюта, пароль админки.

Если такой объём подходит, можно переходить к реализации по шагам (сначала сервер + меню, потом главная, отзывы, галерея, тексты, настройки).
